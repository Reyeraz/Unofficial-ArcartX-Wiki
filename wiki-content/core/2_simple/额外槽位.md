# 额外槽位
自定义额外槽位指南
对真理而言，信服比流言更危险。
## 额外槽位
  * 槽位配置位于ArcartX根目录中的 `extra_slot` 目录下，你可以创建多个yml配置，只需要遵循以下格式新建槽位即可。
  * 槽位应用于后续讲解的ArcartX的自定义UI中，而脚本功能前面提到过，这里就不再反复讲解了。


## 新建槽位
    
    Slot1: # 配置项名作为ID 不要重复
      attribute: "none" # 槽位属性 当前支持：none(不启用槽位属性)、AttributePlus（启用AttributePlus属性支持 需要AttributePlus3.0+）、AstraXHero
      limit: # 槽位限制，请查看脚本如何使用以了解这里如何填写，无限制可以删除该配置项
        - "检测玩家等级{level = 10}"
      update: # 槽位物品更新后执行的脚本，请查看脚本如何使用以了解这里如何填写，无需执行脚本可以删除该配置项
        - "给玩家发送消息{message = 你更新了你的槽位物品}"
      loadSubstitutionModel: false # 是否加载替换模型功能 (这类似时装，但是需要替换玩家模型为ax的自定义模型才能使用，具体的请看模型章节详解。开启后以传统模式读取物品模型进行时装穿戴。
     
    # 这里做一个放入物品后读取nbt然后根据nbt设置玩家模型来实现一个类似于时装的功能
    Slot2:
      attribute: "none"
      limit:
        - "检测物品包含lore{lore = 模型}"
      # 这里忘了做物品判空移除模型了，我也懒得写了，相信看到这里您也有了一些自食其力的能力了。
      update:
        - "通过NBT设置模型{key = model}"
